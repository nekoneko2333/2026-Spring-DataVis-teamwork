## 问题一：播放按钮不能播放 —— 复测结论：此前已修复，本轮复测通过

### 定位过程

- 首页（`data-page="showcase"` + `story-mode`）会把 `#topbar / #left / #right / #bottom` 全部 `display:none`，所以**首页上根本没有时间控制和 `#btnPlay`**，首页的「播放」实际上是 hero 按钮「播放生成短片」（实时 story）。
- 真正的时间轴播放按钮在「探索」页才出现。

### 根因其实已被解决

- 该 bug 的真实原因是：当 `state.step` 停在最后一步（如短片结束后 `_finishCinematic()` 会 `setStep(99)`）时，再点播放，`play()` 置 `playing=true` 后 `_tick()` 立即因 `step>=last` 调 `pause()`，看起来「点了没反应」。
- 这一点已在 commit **`ffb9362`（mcf：debug完成）** 修复——`play()` 里加入了：

  ```js
  const last = state.meta.timeSteps - 1;
  if (state.step >= last) this.setStep(0);   // src/main.js play()
  ```

- 时间线证据：`621mcf修改.md` 最后一次改动在 `5f81e9d`，而 `ffb9362` 在其**之后**，所以文档里「待 debug」是在修复之前写的，属于过期描述。

### 本轮自动化复测（全部 PASS）

| 路径 | 结果 |
|---|---|
| 冷启动 → 进入探索 → 从 t0 播放 | `0 → 45` ✅ |
| 停在末步 t99 再点播放（应回到起点继续放） | `99 → 33` ✅ |
| 完整播完短片 → complete 状态 → 进入探索 → 播放 | `99 → 36` ✅ |
| 首页 hero「播放生成短片」 | story 正常推进、章节切换 ✅ |
| 真实 DOM `#btnPlay.click()` 事件链 | 步进推进 ✅ |

控制台无报错。结论：**播放功能现状正常，本轮不改动相关代码。**

### 关于「首页没有时间控制」的设计说明

首页隐藏普通时间控制是刻意的沉浸式设计，首页的时间推进交给「短片」。如果后续希望首页也能像探索页那样手动拖动/播放时间轴，那是一个独立的产品决策（需要在 showcase 布局里保留 `#left` 的播放条），不属于本次 bug 修复范围，这里只记录，不擅自加 UI。

## 问题二：短片「片层坍缩」基本看不到 —— 已修复

### 根因（实测定位，非猜测）

短片播放时 `#viewport` 背景是**黑色**（`#app[data-page="showcase"] #viewport{background:#000}` 与 `story-mode` 的浅色渐变同特异度、且写在后面，覆盖为黑底）。在黑底 + 灰阶传递函数下：

- 灰阶配色只有 **q90 以上**才有明显亮度（q75≈`#1f1f1f` 近黑，q90≈`#6e737a`，q95≈`#aeb4ba`）。即只有最高密度的「骨架」才看得见。
- 原 STAGE 02 的刷选区间是 **`[q25, q90]` + boost 5.5**：它**恰好把最亮的高密骨架（>q90）排除掉了**，只留下 q25–q90 这段在灰阶里近黑的中低密度 →叠加出来是一团又黑又糊的东西，于是「片层坍缩基本看不到」。

截图对比（step 26，黑底）：
- 旧 sheet 看法（`[q25,q90]`，boost 5.5）：几乎全黑、无结构。
- 不刷选参考：宇宙网清晰可见。
- 把刷选**上移到有灰度亮度的区间**后：墙/片层成片可见。

### 修改内容（仅动短片 STAGE 02，保持黑白灰、不引入彩色）

文件 `src/main.js`：

1. `_applyStoryLook()` 里 story 专用的 `sheet` 刷选区间
   - 由 `[percent("25"), percent("90")]` 改为 **`[percent("50"), percent("95")]`**。
   - 含义：抬到有灰度亮度的中高密度段，墙/片层才看得见，同时排除最亮的节点（>q95），避免和后面「节点增长」章节抢视觉。
   - 该 `ranges` 对象只在 story 流程里用，**不影响**探索页底部的快捷刷选（那套仍是 `[q25,q75]`）。

2. STAGE 02 / SHEETS 章节 look
   - `density` 由 `1.30` 调到 **`1.18`**；`brushBoost` 由 `5.5` 调到 **`2.8`**。
   - boost 5.5 会把整段中低密度顶到近不透明 → 糊成一团；2.8 配合上移后的区间，得到「能看清内部起伏的连续墙面」而不是黑块。

### 验证

- 用选定参数在章节区间两端 step 16 / 26 / 36 分别截图：墙/片层结构**全程清晰可见**。
- 跑**真实短片**到「片层坍缩」章节整页截图（带 cinematic overlay、用章节自身的 drift→thread 运镜）：可见成片的灰色墙状结构 + 拓扑网点叠加，已不再是黑屏。
- 附带收益：新区间通过的体素更少（q50–q95 ≈45% vs 原 q25–q90 ≈65%）、boost 更低，触发体阴影的采样点减少，**该章节的渲染开销反而下降**。
